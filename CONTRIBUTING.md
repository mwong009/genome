## How to contibute to the project

#### Did you find a bug?

* Ensure the bug was not already reported by searching under Issues.

* Write **complete** and **informative** report on the problem or specific problem. The more well written the report, the easier and faster it can be resolved.

You'll probably need to include these in your issue ticket:

1. What were you doing?
2. What did you expect would happen?
3. What actually happened?
4. Operating system? Python version? Genome version?

#### Are you planning a new feature or a new idea?

* Be sure to read the README.md first to ensure your request is compatible with the high level design goals of the project.
* If you are unsure, you can start a new discussion under Issues.
* If not, open a new pull request and describe what your new feature does. Include the relevant issue number if applicable.

#### Did you fix typos, whitespaces, or syntax formatting?

* Submit a pull request and allow some time to review your changes

#### Do you plan to write documentation?

* Help in writing documentation is always welcomed. You can submit new documentation into the `docs` subfolder by opening a new pull request.
* When you are done be sure to include a link to the wiki page.
